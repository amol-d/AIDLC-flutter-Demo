# Product requirement documents

Filled PRD/BRD documents live here — one file per shippable feature, following the
[PRD/BRD template](../../Documentation/templates/PRD-template.md).

Ingest one by opening a GitHub issue:

> `@codex implement the PRD at docs/prd/<slug>.md`

Keeping PRDs in the repo versions each requirement alongside the code and tests it produced.
